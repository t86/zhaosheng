# 学校页上海报考风险卡设计

## 目标

在学校详情页新增一个统一的“上海报考风险卡”区块，把当前站内已经核到的上海公开口径、特色项目提醒和填报前必须回原文核对的事项集中展示出来。

## 背景

当前学校页已经有三类高价值信息，但它们分散在不同模块里：

- 上海考试院 `2021-2025` 院校专业组线与学校官网公开线
- 特色班型/荣誉方向的官方来源、项目标签和培养说明
- 上海官方填报规则与“填报前核对项”

对于上海目标 `985+` 的家庭来说，这些信息缺的不是“更多内容”，而是“先看哪一层、哪些字段现在已经核到、哪些字段必须回招生章程确认”。

## 设计原则

- 只展示站内已核到的事实层，不补猜测字段。
- 不把“学校线”“类别线”“专业组线”混写成“专业最低分”。
- 不伪造具体的选科、单科、学费、体检值；若站内只核到“有这类要求的官方文件”，页面只提示“必须回原文核对”。
- 风险卡本质是“信息边界声明 + 决策入口”，不是替用户生成最终志愿建议。

## 页面方案

风险卡放在学校页靠前位置，位于“公开报告快照”之后、“最强专业排行榜”之前，保证家长先看到决策层信息，再往下看专业内容。

区块包含三层：

1. 状态摘要
   - 展示 `上海口径较完整 / 上海口径需混合解读 / 上海口径待补`
   - 用一句话说明当前学校在上海公开数据上属于哪类情况
   - 辅助标签展示是否包含精确组线、阈值、Q 组、学校官网补口径、联合培养、校内选拔等

2. 当前已核到的风险信号
   - 每张卡只讲一个事实点，并给出原始来源链接
   - 典型信号包括：
     - 上海考试院已公开院校专业组线
     - 高分段仍有阈值口径
     - 存在 Q 组或单独公布组别
     - 学校官网还有补充分数口径
     - 特色项目已核到上海批次要求文件
     - 中外合作或异地培养项目要单独核
     - 不是所有强班都能在普通批直接锁定
     - 部分项目天然偏长周期深造

3. 填报前必须回原文核对
   - 直接复用上海官方规则层里的 4 项核对清单：
     - 招生章程关键限制
     - 选科是否匹配
     - 位次和计划数
     - 2024 专业录取数据

## 数据设计

新增一个纯函数数据层 `lib/school-risk-card.ts`，输入学校页已有数据：

- 学校 `slug / name / featuredTracks`
- 上海考试院专业组线
- 学校官网上海公开线

输出统一的 `SchoolRiskCard`：

- 状态标签与颜色
- 摘要标题与说明
- 风险标签
- 风险信号数组
- 官方核对清单
- 上海官方链接

风险信号的来源优先级：

1. 上海考试院公开记录
2. 学校官网上海公开线
3. 学校特色项目的官方来源与项目标签
4. 上海官方通用核对规则

## 非目标

- 这轮不补平均分位次、在沪计划数、专业层最低分
- 这轮不解析 PDF 里的具体科目组合值
- 这轮不做个性化“冲稳保”判断

## 验证

- 为风险卡数据层补单元测试，覆盖：
  - 无上海数据时的回退文案
  - 存在考试院精确组线/阈值/学校补口径时的组合输出
  - 中外合作、校内选拔、长周期培养等项目信号
- 运行 `pnpm lint`
- 运行 `pnpm build`
