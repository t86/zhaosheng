# 强基专题头部学校扩充 Design

## Goal
把现有“强基与综评”专题里的强基部分，从 `复旦 / 武大 / 东南` 三张旧案例，升级成更适合上海高分家庭阅读的 `清北华五头部学校强基判断页`。

## Positioning
- 不是完整的全国强基数据库。
- 不是 2026 已定稿的招生简章汇编。
- 是以 `2025 官方完整口径` 为主、明确标注 `2026 待更新` 的头部学校强基判断层。
- 重点服务对象是 `目标清北华五和顶尖 985` 的上海家庭。

## Scope
本轮只扩充现有 [selection page](/Volumes/WORK/codex-guide/life/app/selection/page.tsx) 的强基部分，不新开强基独立路由：
- 保留现有强基总览、模式、算法和适合谁的内容骨架。
- 新增 `清北华五强基速览矩阵`。
- 新增 `2025 官方完整口径 / 2026 待更新` 状态提示。
- 把案例区改成 `重点案例`，优先保留和强化：
  - 复旦大学
  - 清华大学
  - 上海交通大学
- 视公开材料稳定性，再补：
  - 北京大学
  - 浙江大学
  - 南京大学
  - 中国科学技术大学

## Out of Scope
- 不新建来源页。
- 不接入全国所有 985 强基学校。
- 不承诺 2026 规则已经公布。
- 不把公众号解读直接写成事实，必须尽量回到学校招生网或官方简章口径。

## Why This Change
当前强基专题的问题不是“没有信息”，而是：
- 案例过旧，且头部学校覆盖不够。
- 对上海高分家庭来说，真正最关心的是 `清北华五怎么比`，而不是先看武大、东南。
- 页面里没有明确区分 `2025 官方完整口径` 和 `2026 待更新`，会影响信任感。

## Information Architecture

### 1. 强基板块头部状态条
放在现有强基总览区和案例区之间。

内容固定为两层：
- `2025 官方完整口径`
- `2026 待更新`

用途：
- 明确时间边界。
- 降低“是不是旧信息”的不信任。

### 2. 清北华五强基速览矩阵
放在案例区前面，作为新的第一层信息。

学校范围：
- 清华大学
- 北京大学
- 复旦大学
- 上海交通大学
- 浙江大学
- 南京大学
- 中国科学技术大学

每校固定字段：
- 学校名
- 时间口径标签
- 招生专业范围
- 入围方式
- 校测组成
- 破格规则
- 一句话判断

用途：
- 让家长先横向比较头部学校差异。
- 不要求一开始就阅读完整时间线。

### 3. 重点案例区
案例区从“复旦、武大、东南三张 2025 图”改成“重点案例”。

优先保留 3 所：
- 复旦大学
- 清华大学
- 上海交通大学

原因：
- 复旦：出分前笔试 + 复试，规则辨识度高。
- 清华：头部学校典型口径，用户必须看。
- 上交：上海家庭高频关注，地方相关性最强。

案例字段继续沿用现有 `GuideTimelineCase` 风格：
- 学校
- 年份
- 副标题
- 关键时间节点
- 专业/规则信息卡
- 综合成绩公式
- 备注

### 4. 头部学校补充说明
在案例区后增加一小组“补充说明卡”，承接未展开完整时间线的学校：
- 北大
- 浙大
- 南大
- 中科大

内容偏摘要，不展开完整步骤。

## Content Rules
- 所有学校都必须明确标注 `2025 官方完整口径` 或 `2026 待更新`。
- 公众号内容只能作为线索，最终落地表述优先采用学校官网、招生网、强基简章。
- 如果某个学校只核到培养方案或转段信息，但没核到完整强基简章，不写成“完整案例”，只能进摘要补充。
- 不用“最强”“必赢”“更香”这类营销式结论。
- 一句话判断要服务决策，例如：
  - “更适合竞赛基础强、能接受出分前笔试的家庭”
  - “更适合希望先看高考分再决定是否参加校测的家庭”

## Data Model Changes
优先复用 [data/selection-guides.ts](/Volumes/WORK/codex-guide/life/data/selection-guides.ts) 现有 `qiangji` 数据结构，并做最小扩展：

新增建议字段：
- `statusPanels`
  - 用于显示 `2025 官方完整口径 / 2026 待更新`
- `headlineSchoolCards`
  - 用于清北华五速览矩阵
- `focusSchoolCases`
  - 从原 `schoolCases` 里拆出重点案例
- `headlineSchoolNotes`
  - 用于承接北大、浙大、南大、中科大的摘要补充

## UI Changes
主要改动 [app/selection/page.tsx](/Volumes/WORK/codex-guide/life/app/selection/page.tsx)：
- 在 `qiangji` 总览后插入状态条。
- 在案例区前插入头部学校速览矩阵。
- 把案例区标题从“复旦、武大、东南三张 2025 图”改成“重点案例”。
- 案例区只优先展示复旦、清华、上交。
- 在案例区后追加摘要说明卡。

样式主要改动 [app/selection/page.module.css](/Volumes/WORK/codex-guide/life/app/selection/page.module.css)：
- 新增速览矩阵卡片样式。
- 新增状态条样式。
- 新增摘要说明卡样式。

## Source Strategy
优先级：
1. 学校本科招生网 / 强基招生简章
2. 学校官方新闻稿 / 官方招生公众号
3. 学校培养方案、转段名单、官方通知
4. 用户提供的公众号文章，仅作线索交叉核对

本轮重点核查学校：
- 复旦大学
- 清华大学
- 北京大学
- 上海交通大学
- 浙江大学
- 南京大学
- 中国科学技术大学

## Risks
- `2026` 简章尚未全部公开，容易误写成“今年规则”。
- 某些学校公开页面更偏培养方案，不等于招生规则。
- 同一学校可能存在 `简章口径` 和 `官方解读文章口径` 表述差异。

控制方式：
- 统一写时间标签。
- 案例与摘要分层，不硬把未核全学校写进完整案例。
- 有争议时优先保守表述。

## Testing
本轮验证重点：
- 现有 selection 页面结构不被打乱。
- 强基板块新增速览矩阵后，手机和桌面仍可读。
- 复旦、清华、上交案例顺序和头部矩阵信息一致。
- 若某校只进入摘要区，不会误显示成完整案例。
